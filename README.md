# Отклик на кредитный оффер — Альфа-Банк × МФТИ

Решение задачи предсказания вероятности согласия корпоративного клиента на кредитное предложение.

## Задача

Бинарная классификация: предсказать, примет ли клиент конкретный оффер по кредитному продукту (бизнес-кредит, возобновляемая кредитная линия или овердрафт).

Метрика: **ROC-AUC**

## Данные

| Файл | Описание |
|---|---|
| `train_apps.csv` | 145 241 заявка с целевой переменной |
| `test_apps.csv` | 36 311 заявок для предсказания |

Данные имеют **temporal split**: трейн охватывает февраль 2024 — июнь 2025, тест — июнь–декабрь 2025. Модель предсказывает будущее, что исключает использование признаков с утечкой времени.

Дисбаланс классов: **6.1% положительных** (8 846 из 145 241).

## Решение

### Ключевое решение: temporal split

Трейн и тест не пересекаются по времени. Обычный Stratified KFold перемешивает данные случайно — модель валидировалась бы на прошлом, обучившись на будущем, что давало бы завышенный OOF AUC (~0.83) при деградации на реальном тесте.

Используется **temporal validation**: последние 20% трейна по времени выступают валидацией — это честная оценка предсказания будущего. Temporal Val AUC = **0.765**, что соответствует результату на лидерборде.

Признаки на основе даты (`decision_month`, `decision_dow`) не используются: модель никогда не видела месяцы из теста (июнь–декабрь 2025), и такой признак давал ложный прирост на валидации.

### Feature Engineering

Создано 14 новых признаков на основе бизнес-логики:

**Ценовые признаки**

- `rate_spread` = `offered_rate` − `cb_rate` — спред предложенной ставки над ключевой ЦБ. Чем выше спред, тем дороже кредит относительно рынка и тем меньше вероятность согласия.

- `rate_to_cb_ratio` = `offered_rate` / (|`cb_rate`| + ε) — мультипликатор: во сколько раз ставка выше рыночной. Дополняет спред — улавливает нелинейную зависимость при разных уровнях ставки ЦБ.

**Лимитные признаки**

- `loan_to_max_limit` = `loan_amount_last` / (`overdraft_limit_max` + ε) — соотношение запрошенной суммы к максимальному лимиту. Значение > 1 означает, что клиент просит больше, чем банк готов дать.

- `loan_to_min_limit` = `loan_amount_last` / (|`overdraft_limit_min`| + ε) — соотношение запрошенной суммы к минимальному лимиту.

- `dist_to_max_limit` = `overdraft_limit_max` − `loan_amount_last` — расстояние от запрошенной суммы до максимального лимита. Топ-1 по важности: чем больше «запас», тем охотнее клиент соглашается.

- `dist_to_min_limit` = `loan_amount_last` − `overdraft_limit_min` — расстояние от запрошенной суммы до минимального лимита.

- `limit_range` = `overdraft_limit_max` − `overdraft_limit_min` — ширина одобренного диапазона. Широкий диапазон означает гибкость банка в условиях.

- `loan_in_limit_range` = 1 если `overdraft_limit_min` ≤ `loan_amount_last` ≤ `overdraft_limit_max`, иначе 0 — бинарный флаг: попадает ли запрошенная сумма в одобренный диапазон.

**Активность клиента**

- `deb_ul_ratio_30_90` = `sum_deb_ul_30` / (`sum_deb_ul_90` + ε) — тренд переводов юрлицам: отношение за 30 дней к 90 дням. Значение > 0.33 означает, что активность растёт.

- `cnt_ul_ip_ratio_30_90` = `cnt_deb_ul_ip_30` / (`cnt_deb_ul_ip_90` + ε) — тренд количества переводов юрлицам и ИП.

- `digital_engagement` = `count_all_corp_dashboard_events` × `p75_time_spent_minutes` — вовлечённость в интернет-банк. Клиенты, активно пользующиеся банком, лояльнее к его предложениям.

**Финансовое состояние**

- `has_positive_balance` = 1 если `balance_rur_amt_30_min` > 0, иначе 0 — флаг положительного остатка на счёте за последний месяц.

- `turnover_to_balance` = `sum_deb_ul_30` / (|`balance_rur_amt_30_min`| + ε) — оборачиваемость: оборот переводов относительно остатка. Высокое значение указывает на потребность в ликвидности.

- `log_sum_deb_ul_30` = log(1 + max(`sum_deb_ul_30`, 0)) — логарифм суммы переводов за 30 дней. Суммы переводов имеют логнормальное распределение, логарифм убирает влияние выбросов.

- `log_sum_deb_ul_90` = log(1 + max(`sum_deb_ul_90`, 0)) — логарифм суммы переводов за 90 дней.

**Кредитная активность**

- `total_credit_activity_90` = `cnt_deb_loan_90` + `cnt_cred_loan_90` — суммарная кредитная активность за 90 дней: количество выплат и полученных кредитов.

**Профиль клиента**

- `reg_years` = `days_from_authperson_registration` / 365 — срок регистрации управляющего лица в банке, переведённый в годы.

*ε = 1e-9 — малая константа для защиты от деления на ноль*

### Модель

**CatBoost** с temporal split валидацией.

CatBoost выбран за нативную обработку категориальных признаков (`db_group_last`, `fl_adminarea`) и пропусков без дополнительной предобработки.

```python
CatBoostClassifier(
    iterations        = <best_iteration из temporal val>,
    learning_rate     = 0.05,
    depth             = 7,
    l2_leaf_reg       = 3,
    subsample         = 0.8,
    colsample_bylevel = 0.8,
    cat_features      = ['db_group_last', 'fl_adminarea'],
    random_seed       = 42,
)
```

### Результаты

| Метрика | Значение |
|---|---|
| Temporal Val AUC | 0.765 |
| Лидерборд (ROC-AUC × 100) | 75.808 |
